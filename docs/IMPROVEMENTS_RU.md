# Улучшения и план развития

В этом документе описаны потенциальные улучшения, известные ограничения и задачи, извлечённые из кодовой базы.

---

## EventEmitterX

### Улучшения API

- [ ] **Реализовать интерфейс `EventTarget`** — Предоставить полную DOM-совместимую реализацию `EventTarget` для старых версий Node.js. Ссылки на эталонные реализации есть в комментариях кода.
- [ ] **Поддержка `handleEvent` в `removeListener` и `once`** — Опция `supportEventListenerObject` работает только частично. `removeListener` и once-обёртки не полностью поддерживают объекты `{ handleEvent }`.
- [ ] **Вынести `static once()` в отдельный файл** (`eventAwaitFor.ts`) — Текущий `events.ts` содержит ~3800 строк. Перенос `static once()` улучшит поддерживаемость. Требуется двухбранчевая стратегия git для сохранения истории.
- [ ] **Улучшения `events.on()`** — Перенести последние изменения из Node.js: `FixedQueue`, опции `highWaterMark`/`lowWaterMark`, поддержка `Stream#pause`/`resume`.
- [ ] **Переименование `errorEventName`/`stopEventName`** в массивы `error`/`close` для совместимости с Node.js в `eventsAsyncIterator`.
- [ ] **Улучшения `addAbortListener`** — Рассмотреть выравнивание с поведением обхода `e.stopImmediatePropagation()` из Node.js 20+.

### Обработка ошибок

- [ ] **Кастомные классы ошибок** — Создать `EventsAbortError`, `EventsOnceError` со структурированными свойствами (`types`, `errorType`) вместо обычного `Error`.
- [ ] **Интеграция `captureRejections`** — Обеспечить консистентное поведение во всех путях вызова слушателей.

### Производительность

- [ ] **`Map`/`WeakMap` для `_events`** — Рассмотреть использование `Map` вместо простого объекта для хранилища событий, возможно через класс-наследник.
- [ ] **Состояние паузы** — Флаг `EventEmitterX_Flags_paused` определён, но не реализован. Рассмотреть добавление pause/resume для сценариев с backpressure.

### Удобство разработки

- [ ] **Улучшения `eventNames()`** — Возвращать числовые ключи как числа и включать Symbol-ключи через `Object.getOwnPropertySymbols()`.
- [ ] **`kAddListenerAfterDestroyingCallback`** — Заменить `console.warn` на полноценное событие/колбек при добавлении слушателей к уничтоженному эмиттеру.

## EventSignal

### Архитектура ядра

- [ ] **WeakMap-based `_events` во внутренних эмиттерах** — Использовать `WeakMap` в `signalEventsEmitter`, чтобы подписки на сигналы автоматически очищались при сборке мусора.
- [ ] **Кастомный домен/эмиттер на сигнал** — Разрешить переопределение используемого `signalEventsEmitter` через опции конструктора.
- [ ] **События жизненного цикла** — Отправлять события `onCreateEventSignal`, `onDestroyEventSignal`.
- [ ] **Синхронный режим уведомлений** — Сейчас все уведомления подписчиков асинхронные (микротаска). Добавить опцию для синхронных уведомлений.
- [ ] **`batchUpdates`** — Добавить `batchUpdatesStart()`/`batchUpdatesEnd()` для явного контроля над синхронным пакетированием обновлений.

### Управление значениями

- [ ] **`updateReason`** — Добавить второй параметр в `set()` для отслеживания причин обновления. Полезно для истории отладки и для `ignoreUpdateReason` в React-хуках.
- [ ] **`validateSourceValue` / `validateValue`** — Функции валидации в опциях, которые могут отклонять невалидные значения.
- [ ] **Поддержка мутабельных значений в `use()` с редюсером** — Сейчас мутабельные значения (тот же ref объекта), возвращаемые редюсерами, не поддерживаются.

### Вычисляемые сигналы

- [ ] **Исправить отслеживание зависимостей в async computation** — Сейчас сигналы, к которым обращаются после `await` внутри computation, могут быть неправильно отслежены (или может быть отслежен не тот сигнал). Это фундаментальное ограничение, задокументированное в коде.
- [ ] **Улучшение распространения ошибок** — Вместо `console.error`, отправлять ошибки в `signalEventsEmitter` как событие `'signalError'`.
- [ ] **Обход троттлинга для некоторых источников** — Разрешить игнорирование throttle для определённых источников обновлений (`set`, `source`, `trigger`).

### Экшены и методы

- [ ] **Опция `actions` в конструкторе** — Определять методы (экшены) прямо в опциях конструктора, доступные как `signal$.actions.methodName()`.
- [ ] **Опция `views` в конструкторе** — Автогенерация производных EventSignal из функций computation, доступных как `signal$.views.viewName`.
- [ ] **Улучшенный `map()` — обратная связь** — `map()` должен создавать производный сигнал, который может пробрасывать изменения обратно в родительский.

### Интеграция с React

- [ ] **`use()` с deps** — Разрешить передачу массива зависимостей в `use()` для контроля перевычисления редюсера.
- [ ] **Опции `useListener()`** — Добавить опции `suspend`, `noSub`, `ignoreUpdateReason`.
- [ ] **Компонент уничтоженного сигнала** — `_setComponentOnDestroy` сейчас не работает. Реализовать корректный рендеринг для уничтоженного состояния.
- [ ] **Протокол Observable** — Реализовать протоколы [TC39 Observable](https://github.com/tc39/proposal-observable) и [WICG Observable](https://wicg.github.io/observable/).

### Улучшения подписок

- [ ] **Паттерн Observer** — Поддержка объектов-наблюдателей `subscribe({ next, error, complete })`.
- [ ] **`cleanupCallback` в `addListener`** — Третий параметр для очистки слушателя при его удалении.
- [ ] **Автоочистка на основе WeakRef** — `weakSpyOnTarget` для автоматического удаления слушателя при сборке мусора ссылаемого объекта.

### Система типов

- [ ] **Ограничить `componentType`** — Разрешить только `string | number | symbol`, убрать поддержку `Object`.
- [ ] **Унифицировать сигнатуры computation** — `constructor`, `createMethod` и `map` имеют разные сигнатуры функций computation.

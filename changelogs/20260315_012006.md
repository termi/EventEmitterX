## Human-Readable Summary

### EventSignal — New API
- New React hook **`useReducedListener(reducer, listener, options?)`** — reduces the signal value before calling the listener, supports `areReducedValuesEqual` to skip unnecessary calls
- **`useListener`** now accepts `options: { areValuesEqual?, deps? }` (previously `deps` was a positional parameter); supports `areValuesEqual` to suppress re-runs when value hasn't meaningfully changed
- **`destroyOnUnmount`** option in `_reactFC[2]` — when set, the component auto-calls `signal.destructor()` on unmount via `useEffect`
- **`FinalizationRegistry`** integration — signals can register themselves for automatic cleanup when garbage-collected (opt-in via `eventSignalsFinalizationRegistry`)
- **`EventSignalDestroyedComponent`** is now fully enabled — used after `destructor()` is called, renders the last known value without React hooks

### EventSignal — Bug Fixes
- Fixed **`currentSignal` restoration** in async computation: `currentSignal = prev_currentSignal` is now restored immediately after the sync part of computation (before the async branch), and also in the `catch` block — previously only restored in `finally`, which was too late for nested signal reads
- Fixed **`_cDeferredList` data race condition** in the case when the previous computation was
  async and current computation has sync result. Clear array after deferred promises resolved (`_cDeferredList.length = 0`).
  Added test for this scenario: 'handle data racing in hybrid sync/async mode'.
- `onEnd` reference is now cleared alongside `triggerCleanUp` in trigger cleanup callbacks
- `spec_utils/simple-react-hooks.ts`: fixed bug in restarting hooks on re-render

### EventSignal — Refactoring & Performance
- `_checkIsPromise(value)` helper extracted — replaces 6+ inline `!!val && typeof val === 'object' && typeof val["then"] === 'function'` checks
- `setReactFC()` refactored — now handles `null`/`false` (clears FC), array format, `{ 0: fc }` object, and plain function
- `_reactFC` field changed from optional (`?`) to nullable (`| null = null`) for stable V8 hidden class
- Constructor: if computation is an async function, eagerly initializes async-related fields (`_cDeferredList`, `_recalcPromise`, `_resolve`, `_reject`, `status`) to force V8 to pre-compute the final object shape
- Constructor: after setup, resolves `props`, `keyProps`, `displayName` getters to force V8 hidden class finalisation
- `props` getter simplified — removed `Object.setPrototypeOf(..., null)` (V8 doesn't apply hidden classes to null-prototype objects); now uses a plain object for better performance
- `EventSignalComponent` prop renamed: `eventSignal` → `current$` (old name kept as deprecated)
- Ref **`_computation` holding a closure after `destructor()`**: now nulled out to `_noop` on destroy.
  Add todo for `oneTimeComputation` flag support to also clear itself after running once.
- Added `current$Version` and `current$SnapshotVersion` props (old `version`/`snapshotVersion` kept as deprecated)
- `listeners()` / `rawListeners()` return type narrowed from `Function[]` to `((...args: any[]) => void)[]`
- `use()` parameter renamed: `areReducedValueEqual` → `areReducedValuesEqual`
- `tsconfig.json`: removed redundant `ES2025` from `lib` (covered by `ESNext`)
- **`spec/EventSignal_spec.ts`**: fixed `instanceof Promise<string>` → `instanceof Promise`; added `handle data racing in hybrid sync/async mode` test

### EventSignal — Новый API
- Новый React-хук **`useReducedListener(reducer, listener, options?)`** — уменьшает значение сигнала перед вызовом listener, поддерживает `areReducedValuesEqual` для пропуска ненужных вызовов
- **`useListener`** теперь принимает `options: { areValuesEqual?, deps? }` (раньше `deps` был позиционным параметром); поддержка `areValuesEqual` для подавления повторных вызовов
- Опция **`destroyOnUnmount`** в `_reactFC[2]` — компонент автоматически вызывает `signal.destructor()` при анмаунте через `useEffect`
- Интеграция **`FinalizationRegistry`** — сигналы могут регистрироваться для автоматической очистки при сборке мусора
- **`EventSignalDestroyedComponent`** теперь полностью включён — рендерит последнее известное значение без хуков React после вызова `destructor()`

### EventSignal — Исправления ошибок

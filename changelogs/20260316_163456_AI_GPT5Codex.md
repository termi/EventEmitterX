- Added `#### Scope Rules (required)`: AI-config-only commits must use `scope=config/ai`; `docs(config):` is explicitly forbidden for such commits
- Updated `### 2. AI Authorship Prefix` examples to use varied model names instead of hardcoded `ClaudeSonnet4.6`; replaced "Current model name" static value with flexible ModelName description
- Added `scope`/`Area` alignment requirement in `### 5. Required Git Trailers`
- Added rule in `### 7. Staging Files` to split mixed AI-config and non-AI changes into separate commits

**Documentation Rules:**
- Added mandatory frontmatter properties block for all AI-generated Markdown files: `iso date`, `timestamp` (Milliseconds since epoch), `ai_model`, `git user`, `area`
- Clarified `timestamp` field as "Milliseconds since epoch" (not Unix seconds)

**Git Command Rules:**
- Added `### Date/Time and Timestamp Retrieval Rules` with strict fallback order: Node.js → Python → shell
- Added `#### Local date/time according CLDR/ICU` subsection with locale-aware API requirements for Node.js (`toLocaleString()`, `toLocaleDateString()`, `toLocaleTimeString()`) and Python (`locale.localeconv` + `now.strftime`)
- Added `#### Milliseconds since epoch` subsection with concrete implementations: `Date.now()` in Node.js, `int(time.time() * 1000)` in Python, `date +%s%3N` in shell, with fallback note to multiply Unix timestamp × 1000

**Changelog Structure:**
- Added optional `## Original User Request` section
- Added optional `## Change Assessment` section
- Expanded RU-translation sections with individual headings
- Updated changelog title format to `Local datetime according CLDR/ICU`

**Technical Review Rules:**
- Added entire new section `## Technical Review Rules` requiring: file path + line numbers + code snippet for every issue, severity labels (🔴🟠🟡🔵), Recommendation subsection per issue, structured issue format

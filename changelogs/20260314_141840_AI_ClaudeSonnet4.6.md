## Technical Details

### File changed: `.github/copilot-instructions.md`

Four separate additions were made to the Copilot instructions file:

**1. No-auto-commit warning (Section intro)**
Added a prominent `⚠️ Important` warning at the top of the Git Commit Conventions section,
explicitly stating that the AI must NOT commit changes on its own initiative — only when
the user explicitly requests it.

**2. Section 8 — Human-Made Commits**
Added a new rule section that defines behavior when the user indicates changes were made
by themselves (e.g. "закоммить мои изменения", "commit my changes"):
- Skip the `(AI/<ModelName>):` prefix in the commit subject
- Do NOT add Git Trailers unless explicitly requested
- All other commit rules (Conventional Commits, English, structure) still apply

**3. Documentation Language Rules (new top-level section)**
Added rules for documentation files:
- Default language is English
- A `_RU` suffixed Russian version must always be created alongside (e.g. `API_RU.md`)
- Both files must have equivalent content, structure and detail level
- Both must be committed together under the `docs` type
- Exception: no `_RU` counterpart for files in AI-tooling config folders
  (`.github/`, `.cursor/`, `.continue/`, etc.)

**4. Changelog Strategies (new top-level section)**
Added rules for auto-generating changelog files in the `changelogs/` folder:
- Defined a shared changelog file structure (header, meta, human-readable summary,
  technical details, Russian duplicate)
- Defined three triggers with a comparison table:
  - AI-made changes → `<datetime>_AI_<ModelName>.md` (includes Model + AI reasoning)
  - User requests changelog for own changes → `<datetime>.md` (no AI meta)
  - User requests commit for own changes and no uncommitted changelog exists → `<datetime>.md`

**5. Git Command Rules (new top-level section)**
Added a mandatory rule for AI agents when calling git commands:
- Commands with paginated output (`log`, `diff`, `show`, `blame`, `shortlog`) must always
  be called with the `--no-pager` flag (e.g. `git --no-pager log`)
- Without this flag the terminal will hang waiting for input in an automated context

# Как работает интернационализация

При смене локали для строк, для которых нет закешированных или заранее созданных переводов, вызывает **неофициальное**
API `translate.googleapis.com/translate_a/single` для перевода строки с 'ru-RU' на нужную локаль.

## Ограничения (сгенерировано DeepSeek)
Да, использование неофициального конечного адреса `translate.googleapis.com/translate_a/single` действительно сопряжено
с серьёзными и непредсказуемыми ограничениями, которые носят, в первую очередь, защитный характер, а не являются
публичной услугой.

### ⚠️ Ключевые ограничения неофициального метода
Вот основные проблемы, с которыми вы можете столкнуться:

| Ограничение | Описание |
| :--- | :--- |
| **Отсутствие документации** | Этот адрес не является частью публичного **Google Cloud Translation API**. Его поведение, параметры и лимиты не задокументированы Google для разработчиков и могут измениться в любой момент. |
| **Частые блокировки** | Наиболее распространённая проблема — получение ошибки **HTTP 429 ("Too Many Requests")**. Google активно блокирует запросы, которые система определяет как "необычный трафик" или "автоматизированные запросы", особенно с одного IP-адреса. |
| **Зависимость от IP-адреса** | Лимиты часто применяются на уровне IP-адреса, а не API-ключа. Это делает метод крайне ненадёжным для серверных приложений, где все запросы идут с одного адреса. |
| **Прерывистые отказы в работе** | Вы можете успешно отправлять запросы, а затем внезапно начать получать ошибки 429 без видимой причины. Блокировка обычно временная, но её длительность непредсказуема. |

### 🔗 Сравнение с официальным API
Для сравнения, официальное **Google Cloud Translation API** (платное) имеет прозрачные, настраиваемые квоты, которые
вы контролируете в своём проекте Google Cloud.

| Параметр | Неофициальный метод (`translate_a/single`) | Официальный Cloud Translation API |
| :--- | :--- | :--- |
| **Статус** | Не поддерживается, недокументировано | Публичная, коммерческая услуга |
| **Ограничения** | Неизвестны, применяются автоматически и агрессивно | Известны, настраиваемые (напр., 6 млн символов в минуту) |
| **Надёжность** | Очень низкая, частые блокировки | Высокая, гарантируется соглашением об уровне услуг (SLA) |
| **Поддержка** | Отсутствует | Официальная техническая поддержка |

### 💡 Рекомендации и альтернативы
Если вы использовали этот метод для прототипа, вот что стоит учесть:

1.  **Для продакшена или важных задач** следует перейти на официальный **Google Cloud Translation API** (у него есть бесплатный ежемесячный лимит).
2.  **Если вам принципиально нужен бесплатный метод**, будьте готовы к нестабильности. Попробуйте добавить в ваш Node.js
   сервер искусственные задержки (`setTimeout`) между запросами (например, 1-2 секунды) и реализовать обработку
   ошибки 429 с повторной попыткой через некоторое время. Это может немного снизить вероятность блокировки, но не устранит её полностью.

Для вашего текущего приложения выбор зависит от требований к надёжности и бюджета.
Если вам нужны стабильные переводы без постоянного страха блокировки — официальный API будет самым правильным решением.
